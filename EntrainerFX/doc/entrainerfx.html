Entrainer FX Lite

A stripped down version of Entrainer FX - EntrainerFX Lite - is available for use on the web. A work in progress.

Entrainer FX Features - CSS control of the GUI

The borders, the font, the colours and most strikingly the new background picture are all controlled by JavaFX's ability to be configured via CSS. This file resides as EntrainerFX/css/entrainer.css and is editable with any good text editor. Those who do not know CSS may only wish to change the background picture. This is accomplished by copying the picture to the css directory, modifying the following in the entrainer.css file and restarting Entrainer:

#background-image {
-fx-image: url("my-new-picture.jpg");
}

The images included with the Entrainer distribution are from Free Nature Pictures and from National Geographic Science Photos

Further understanding of CSS usage in JavaFX can be obtained from the JavaFX/CSS reference or via tutorials.



Entrainer FX Features - Titled Panes

This paradigm is one I had wished Swing supported when I initially coded Entrainer. Clicking on the appropriate bar causes the titled pane to expand, revealing controls for pink noise panning, flashing, animations and shimmer effects. Clicking the bar again causes the titled pane to collapse and become semi-translucent. This replaces the need in Swing for pop up dialogs.



Entrainer FX Features - Pure Java JSyn

As the implementation of JavaFX animations greatly reduced the clicking I've opted to only include the pure Java JSyn library. This version of the library can interface with PortAudio which may eliminate any platform issues entirely. As this is alpha software no guarantees are provided. Should any audio problems manifest deleting the jportaudio.jar file under the lib directory will default the JSyn sound library to use the JavaSound API. Updates to this interface and the JSyn library itself can be downloaded from the JSyn library download page.



Java FX Impressions - EntrainerFX Rewrite

The Good

  1. Much better animation support
  2. Titled panes, a much needed paradigm
  3. CSS configuration, a big plus
  4. Mature and intuitive API (+1 to event handling)
  5. Use of degrees instead of radians
  6. JavaFX integration with Swing is a breeze

The Not So Good

  1. Missing a spinner control (under development in JFXtras) - prevented further rework for this release

The Ugly

  1. The building and deploying of JavaFX applications

As a consequence of the rewriting of the animation subsystem the animation code is now dependent on javafx.geometry.Rectangle2D. Building and deploying Entrainer using the recommended Ant task results in an application that cannot dynamically load EntrainerFX animations - a NoClassDefFound exception is thrown for Rectangle2D. Including the JavaFX runtime jar file in the distribution and building without the new Ant task gets around this.

But I should not have to do this. If Oracle wishes JavaFX to replace Swing then JavaFX must become a first class citizen of the JRE. I trust this will be fixed in future releases of JavaFX: something like -XX:+EnableJavaFX

After spending some time on a completely unrelated task into user inputs during installation I decided to make full use of the api I liberated.
